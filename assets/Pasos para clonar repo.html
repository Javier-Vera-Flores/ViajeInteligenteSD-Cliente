S, un programa en Arduino siempre tiene al menos dos funciones principales:

  1. setup():
    Esta funcin se ejecuta una sola vez cuando la placa Arduino se inicializa o se reinicia. Su propsito es configurar los pines, inicializar variables, establecer comunicaciones, y cualquier configuracin inicial necesaria para el programa.

    Esta funcin contiene el cdigo que se ejecutar repetidamente mientras la placa est encendida. Es donde defines las acciones que el Arduino debe realizar continuamente.

Estructura bsica de un programa Arduino

Aqu tienes un ejemplo bsico:

  // Cdigo de configuracin inicial
  Serial.begin(9600); // Inicia la comunicacin serial
  pinMode(13, OUTPUT); // Configura el pin 13 como salida
}

void loop() {
  // Cdigo que se ejecuta repetidamente
  digitalWrite(13, HIGH); // Enciende el LED en el pin 13
  delay(1000);            // Espera 1 segundo
  digitalWrite(13, LOW);  // Apaga el LED

  1. El flujo del programa:

    • Al iniciar el Arduino, primero se ejecuta setup().
    • Luego, loop() comienza a ejecutarse en un bucle infinito.

  2. Comportamiento de setup():

    • Solo se ejecuta una vez, sin importar cunto tiempo dure el programa.
    • Es til para configuraciones como:
      • Configurar pines como entrada o salida.
      • Iniciar una conexin serial.
      • Inicializar libreras.

  3. Comportamiento de loop():

    • Despus de que termine una ejecucin de loop(), Arduino automticamente vuelve a ejecutarla desde el inicio.
    • Es donde se coloca el cdigo para leer sensores, controlar actuadores, y realizar tareas repetitivas.

Qu pasa si no incluyes loop() o setup()?

  • Sin loop(): El programa compilar, pero no har nada despus de ejecutar el cdigo en setup().
  • Sin setup(): Es obligatorio, aunque no hagas nada en ella (puede estar vaca). Ejemplo:
